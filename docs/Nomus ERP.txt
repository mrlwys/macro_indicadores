Nomus ERP
API do Nomus ERP
Nossas APIs necessitam da chave-integração-rest que pode ser obtida através da tela de configuração geral do seu
sistema.
Todos os endpoints de listagem podem ser paginados através do parâmetro "pagina" que tem como padrão o valor 1.
Além disso, todos os endpoints de listagem podem ser filtrados através do parâmetro "query" seguindo o padrão:
?query=nomeCampo=valorCampo;nomeCampo2=valorCampo2
Os campos disponíveis para filtragem de um endpoint dependem dos campos que são retornados em uma requisição
àquele endpoint.
Para filtragem em campos data: ?query=campoData>yyyy-mm-ddTHH:mm:ss

Email: suporte@nomus.com.br
Throttling
Com o objetivo de evitar o uso excessivo da API temos um mecanismo de throttling que bloqueia qualquer requisição
após uma certa quantidade total de requisições seja ultrapassada. Quando isso acontecer em vez do retorno
esperado da API será retornado uma respostas que indica o bloqueio e possui statuscode 429 �TOO MANY
REQUESTS�.
Requisições a nossa API sempre deve tratar esse possível retorno através do statuscode. Além disso, o integrador
deve implementar esse tratamento levando em consideração o campo tempoAteLiberar, que em segundos, vai indicar
quanto tempo deve ser aguardado até uma próxima requisição.
Como preparar meu sistema para o throttling?
Existem diversas formas de implementar no seu sistema a capacidade de aguardar o nosso throttling. O modo mais
usual e simples é verificar se o código da resposta HTTP foi o 429, e se for verdadeiro, implementar a capacidade de
aguardar pelo tempo que enviamos no campo "tempoAteLiberar" da resposta e refazer a sua requisição novamente
em seguida.
